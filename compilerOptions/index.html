# TypeScript Compiler Options

# TS 配置文件

# 问题

  1. 一个比较老的 commonjs 库由于没有默认输出，采用的是 module.exports 的输出方式，导致使用 import xxx from xxx 的方式失效，TS 在配置上 esModuleInterop 及时在解决这个问题，如果遇到的库类有这种问题，直接修改 esModuleInterop 为 true 即可。同时一般使用 allowSyntheticDefaultImports:true 来允许从没有默认导出的模块进行默认导入，或者使用 skipDefaultLibCheck:true 来跳过文件的声明文件检查

# 命令行编译器选项

序号 编译选项 默认值 说明
1 allowJs false 是否允许解析 JavaScript
2 allowSynceticDefaultImports module === "system" or --esModuleInterop 允许从模块进行默认导入而没有默认导出。这不影响代码发出，仅影响类型检查
3 allowUmdGlobalAccess false 允许从模块访问 UMD 全局变量
4 allowUnreachableCode false 不要报告无法访问的代码错误
5 allowUnusedLabels false 不要在未使用的标签上报告错误。
6 alwaysStrict false 以严格模式解析并"use strict"为每个源文件发出
序号 编译选项 默认值 说明
7 assumeChangesOnlyAffectDirectDependencies false 在“–incremental”和“–watch”中进行重新编译时，假设文件中的更改仅会直接影响与其相关的文件
序号 编译选项 默认值 说明
8 baseUrl false 基本目录，用于解析非相对模块名称。有关更多详细信息，请参见模块分辨率文档
9 build false 生成此项目及其由Project References指定的所有依赖项。请注意，此标志与此页面上的其他标志不兼容。在这里查看更多
10 charset utf8 输入文件的字符集

--checkJs 默认值："boolean"

报告.js文件中的错误。与结合使用--allowJs。

# --composite

确保TypeScript可以确定在哪里可以找到引用项目的输出以编译项目。

# --decalaration Or -d

生成相应的.d.ts文件。

# --decalarationDir

生成的声明文件的输出目录。

# --declararationMap

为每个相应的“ .d.ts”文件生成一个源映射。

# --diagnostics

显示诊断信息。

# --disableSizeLimit

禁用JavaScript项目的大小限制。

# --downlevelIteration

for..of针对ES5或ES3时，可扩展，扩展和销毁中的可迭代项提供全面支持。

# --emitBOM

在输出文件的开头发出UTF-8字节顺序标记（BOM）。

# --emitDeclarationOnly

仅发出“ .d.ts”声明文件。

# --emitDecoratorMetadata

发出设计类型的元数据，以在源代码中进行修饰的声明。有关详细信息，请参见问题2577。

# --esModuleInterop

发出__importStar和__importDefault运行时Babel生态系统兼容性的帮助程序--allowSyntheticDefaultImports，并实现类型系统兼容性。

# --experimentalDecorators

为ES装饰器启用实验性支持。

# --extendedDiagnostics

显示详细的诊断信息

# --forceConsistentCasingInFileNames

禁止对同一文件使用大小写不一致的引用。

# --generateCpuProfile

在给定路径下生成cpu配置文件。传递现有的目录名称而不是文件路径将导致在该目录中生成以时间戳命名的配置文件。

# --help or -h

打印帮助信息。

# --importHelpers

进口EMIT佣工（例如__extends，__rest等..）从tslib

# --incremental

通过将以前的编译信息读/写到磁盘上的文件来启用增量编译。该文件由--tsBuildInfoFile标志控制。

# --inlineSourceMap

发出带有源映射的单个文件，而不是拥有单独的文件。

# --inlineSources

在单个文件中将源代码与源映射一起发射；要求--inlineSourceMap或--sourceMap要设置。

# --init

初始化TypeScript项目并创建tsconfig.json文件。

# --isolatedModules

执行其他检查，以确保单独的编译（例如使用transpileModule或@ babel / plugin-transform-typescript）是安全的。

# --jsx

支持JSX的.tsx文件："react"，"preserve"，"react-native"。参见JSX。

# --jsxFactory

指定在响应React JSX发出时要使用的JSX工厂函数，例如React.createElement或h。

# --keyofStringsOnly

仅解析keyof为字符串值的属性名称（无数字或符号）。

# --useDefineForClassFields

使用ECMAScript标准语义发出类字段。

# --listEmittedFiles

在编译过程中打印生成文件的名称。

# --listFiles

打印文件名称的一部分。

# --locale

用于显示错误消息的语言环境，例如en-us。 可能的值为： ►英语（美国）：en ►捷克语：cs ►德语：de ►西班牙语：es ►法语：fr ►意大利语：it ►日语：ja ►韩语：ko ►波兰语：pl ►葡萄牙语（巴西）：pt-BR ►俄语：ru ►土耳其语：tr ►简体中文：zh-CN ►繁体中文：zh-TW

# --mapRoot

指定调试器应定位地图文件的位置，而不是生成的位置。如果.map文件在运行时位于与.js文件不同的位置，请使用此标志。指定的位置将嵌入到sourceMap中，以将调试器定向到地图文件所在的位置。该标志将不会创建指定的路径，也不会在该位置生成地图文件。而是，创建一个将文件移动到指定路径的构建后步骤。

# --maxNodeModuleJsDepth

在node_modules下搜索并加载JavaScript文件的最大依赖深度。仅适用于--allowJs。

# --module Or -m

指定模块代码生成："None"，"CommonJS"，"AMD"，"System"，"UMD"，"ES6"，"ES2015"或"ESNext"。 ►只有"AMD"和"System"可与结合使用--outFile。 ► "ES6"和"ES2015"可以在定位"ES5"或更低时使用值。

# --moduleResolution

# 确定如何解决模块。可以"Node"使用Node.js / io.js样式解析"Classic"。有关更多详细信息，请参见模块分辨率文档

# --newLine

发出文件时，请使用指定的行尾顺序："crlf"（windows）或"lf"（unix）。”

# --noEmit

不要发射输出。

# --noEmitHelpers

不要像__extends在编译输出中那样生成自定义帮助程序函数。

# --noEmitOnError

如果报告了任何错误，请不要发出输出。

# --noErrorTruncation

不要截断错误消息。

# --noFallthroughCasesInSwitch

在switch语句中报告失败情况的错误。

# --noImplicitAny

在隐any式类型的表达式和声明上引发错误。

# --noImplicitReturn

当函数中的所有代码路径均未返回值时，报告错误。

# --noImplicitThis

在this隐any式类型的表达式上引发错误。

# --noImplicitUseStrict

不要"use strict"在模块输出中发出指令。

# --noLib

不包括默认的库文件（lib.d.ts）。

# --noResolve

不要将三斜杠引用或模块导入目标添加到已编译文件的列表中。

# --noStrictGenericChecks

禁用对函数类型中通用签名的严格检查。

# --noUnusedLocals

报告未使用的本地错误。

# --noUnusedParameters

报告未使用参数的错误。

# --outDir

将输出结构重定向到目录。

# --outFile

连接并输出到单个文件。串联顺序由命令行上传递给编译器的文件列表以及三斜杠引用和导入确定。有关更多详细信息，请参见输出文件订购文档。

# paths

路径映射条目列表，用于模块名称到相对于的位置baseUrl。有关更多详细信息，请参见模块分辨率文档。

# --preserveConstEnums

不要在生成的代码中擦除const枚举声明。有关更多详细信息，请参见const枚举文档。

# --preserveSymlinks

不要将符号链接解析为其真实路径；将符号链接文件视为真实文件。

# --preserveWatchOutput

将过时的控制台输出保持在监视模式下，而不是清除屏幕

# --pretty

使用颜色和上下文样式化错误和消息。

# --project Or -p

给定有效配置文件的项目。 该参数可以是有效JSON配置文件的文件路径，也可以是包含该tsconfig.json文件的目录的目录路径。 有关更多详细信息，请参见tsconfig.json文档。

# --reactNamespace

已弃用。使用--jsxFactory代替。 指定针对JSX发出createElement并__spread针对其"react"发出的对象。

# --removeComments

删除所有以开头的版权标题标题注释除外的注释 /*!

# --resolveJsonModule

包括随.json扩展名导入的模块。

# --rootDir

指定输入文件的根目录。仅用于通过来控制输出目录结构--outDir。

# rootDir

根文件夹列表，其组合内容表示运行时项目的结构。有关更多详细信息，请参见模块分辨率文档。

# --showConfig

而不是实际使用其他输入选项和配置文件执行构建，而是在输出中显示最终的隐含配置文件。

# --skipDefaultLibCheck

已弃用。使用--skipLibCheck代替。 跳过默认库声明文件的类型检查。

# --skiplibCheck

跳过所有声明文件的类型检查（*.d.ts）。

# --sourceMap

生成相应的.map文件。

# --sourceRoot

指定调试器应在其中定位TypeScript文件的位置，而不是源位置。如果源在运行时的位置与设计时的位置不同，请使用此标志。指定的位置将嵌入到sourceMap中，以将调试器定向到源文件所在的位置。

# --strict

启用所有严格的类型检查选项。 启用--strict使--noImplicitAny，--noImplicitThis，--alwaysStrict，--strictBindCallApply，--strictNullChecks，--strictFunctionTypes和--strictPropertyInitialization。

# --strictBindCallApply

启用的更严格的检查bind，call以及apply对功能的方法。

# --strictFunctionTypes

禁用函数类型的双变量参数检查。

# --strictPropertyInitialization

确保在构造函数中初始化未定义的类属性。--strictNullChecks必须启用此选项才能生效。

# --strictNullChecks

在严格的null检查模式下，nulland undefined值不在每种类型的域中，并且只能分配给自己，并且any（一个例外undefined是也可以分配给void）。

# --suppressExcessPropertyErrors

禁止检查对象文字的多余属性。

# `--suppressImplicitAnyIndexErrors

禁止--noImplicitAny为缺少索引签名的对象建立索引的错误。有关更多详细信息，请参见问题＃1232。

# --target Or -t

指定ECMAScript的目标版本： ► "ES3"（默认值） ► "ES5" ► "ES6"/ "ES2015" ► "ES2016" ► "ES2017" ► "ES2018" ► "ES2019" ► "ES2020" ► "ESNext"

注："ESNext"最新支持的目标ES建议功能。

# --traceResolution

报告模块解析日志消息。

# --tsBuildInfoFile

指定要在其中存储增量构建信息的文件。

# --types

要包括的类型定义的名称列表。有关更多详细信息，请参见@types，–typeRoots和–types。

# --typeRoots

包含类型定义的文件夹列表。有关更多详细信息，请参见@types，–typeRoots和–types。

# --version Or -v

打印编译器的版本。

# --watch Or -w

在监视模式下运行编译器。监视输入文件并触发对更改的重新编译。监视文件和目录的实现可以使用环境变量进行配置。有关更多详细信息，请参见配置手表。
