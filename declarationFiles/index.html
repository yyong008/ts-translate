# 声明文件

首先搞明白为什么会有声明文件的存在，一个语言好好的，为什么还需要一个声明文件？其实这声明文件并不是为 TypeScript 自己设计的。TypeScript 为了兼容 JavaScript 生态，JS 的代码没有办法获取到很好的获取静态语言的能力，为了补全这个能力，TypeScript 提出声明文件，写好了声明文件，我们就能很平滑的在原有的JS库上书写TS代码，所以TypeScript的声明文件是为类库准备的。

如果我们不编写类库，就是自己的应用，我们不需要使用声明文件，类型直接在代码内部编写就行了。

但是，我们还是要熟悉主流的声明文件，以及它们的编写方式，我们可以书写声明文件是如何编写的，如何组织我们声明文件，如果我们也要写我们自己的库，或者我们要在声明文件上进行增删改查。
