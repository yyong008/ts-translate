# 模块化库

TS 中主要是两个模块化方案：

  • ECMAScript 2015 规范
  • CommonJS 规范

一些书写的变化：

// in CommonJS
var fs = requrie('fs')

// in TypeScript or ES2015
import fs = require('fs')

import 是解构的语法，而 var是赋值的语法。

模块化的库中，一般是不会有 window或者global等全局字样的。模块化中常用的有：

  • require
  • define
  • import * as a from 'b'
  • export x;
  • exports
  • module.exports

# umd 模块化

认识 umd 模块的时候，我们知道会还有三个判断：

首先判断 define 是不是一个函数，其次判断 module 是不是一个 object 同时看 module.exports 是不是存在的就只能判断是是不是 commonJS 模块，然后就是 factory 函数进行处理
