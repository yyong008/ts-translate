# 深入理解

这里的理论是是如何写复杂的定义文件的工具，这些文件暴露出友好的API。这里侧重 模块或者UMD模块。

# 关键概念

类型, 创建一个新的类型

  • 类型别名声明 type sn = number | string
  • 接口声明 interface I {x: number[];}
  • 类声明 class C{}
  • 枚举声明 ```enum E { A, B, C}
  • import 引用类型

# 值

  • let, const 和 var 声明可创建一个值
  • namespace or module 声明轴可以包含值
  • enum 枚举可以包含一个值
  • class 声明包含一个值
  • import 引用值的生命
  • function 声明

命名空间，类型可以存在于命名空间中，例如： 有声明 let x: A.B.C， 则说类型 C 来自 A.B 命名空间。

这种区别是微妙的，在这里， A.B 不一定是类型或者值。

export var SomeVar: {
  a: SomeType
}

export interface SomeType {
  count: number
}

// 使用它
import * as foo from './foo'
let x: foo.SomeType = foo.SomeVar.a
console.log(x.count);

接口和变量是同名的：作为值，和作为类型

export var Bar: {a: Bar}
export interface Bar {
  count: number
}

import { Bar } from './foo'
let x: Bar = Bar.a;
console.log(x.count);

namespace X {
