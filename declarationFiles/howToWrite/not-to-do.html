# 什么不能做

# 类型： 一般类型

  • Number
  • String
  • Boolean
  • Symbol
  • Object

在 JavaScript 中这些都是基本的包装类型

// wrong
function reverse(s: string): String;

// OK

function reverse(s: string): string;

这里的 String 要理解为包装对象，所以包装对象用在这里是不合适的。仅仅使用 string 表示类型就可以了。

# 回调函数

在回调函数中我们不要使用 any，因为 any 不安全，如果不确定我们可以置空使用 void。

// wrong,不要将 any 作为函数的返回值类型
function fn(x: () => any) {
  x();
}

// 正确的做法是：
function fn(x: () => void) {
  x();
}

为什么 any 是不安全的，void 是安全的？any 的不安全是不是有其他的让 any 变的安全。

function fn(x: () => void) {
  var k = x();
  k.doSomething();
}

这个例子中我们将 x 定义成一个函数，这个函数的返回值是空的，但是但是，我们赋值给 k 之后，我们在调用 doSomething 方法，这个时候 ts 的编译器就会检测到， void 怎么还能调用方法呢？这明显不行。所以不要使用 any 作为函数的返回值。

# 回调函数中的参数问题

不要使用可选的参数：

// wrong
interface Fether {
  getObject(done: (data: any, elapsedTimed?: number) => void): void;
}
// OK
interface Father {
  getObject(done: (data: any, elapsedTime: number) => void): void;
}

一个函数 A 中，传入另一个参数 B，该参数 B 是函数（B is funciton），那么？

B 一定是充当回调函数使用的，这就是回调函数的意义。准确的说，一个作为参数的函数的意义，就是充当回调函数的使用。（完后一些任务之后，我们需要调用函数，这个函数就是回调函数）。

// wronh
declare function beforeAll(action: () => void, timeout?: number): void;
declare function beforeAll(action: () => void, timeout?: number): void;

// 正确的写法是：一次尽量将所有的单数写全乎
declare function beforeAll(
  action: (done: DoneFn) => void,
  timeout: number
): void;

# 重载的排序问题

跟很多的排序问题一样，重载也面临着排序的问题， ts 也是在找到第一个重载主体，就不在向下找了。

从一般到具体的原则

我们在使用的时候，更多的是具体的，但是一般性的用的很少。f'f't

// wrong
declare function fn(x: any): any;
declare function fn(x: HTMLElement): number;
declare function fn(x: HTMLDivElement): string;

var myElem: HTMLDivElement;
var x = gn(myElem);

使用可选的参数：

// 可选的参数：
interface Example {
  diff(one: string): number;
  diff(one: string, two: string): number;
  diff(one: string, two: string, three: number): number;
}

// 上面语法上是没有错的，但是我们不应该这么干！
interface Example {
  diff(one: string, two?: string, three?: boolean): boolean;
}

function fn(x: (a: string, b: number, c: number) => void) {}
var x: Example;
fn(x.diff);

var x: Example
x.diff("something", true ? undefined : "hour")

使用联合类型将不必要的重载合并

// wrong
interface Moment {
  utcOffset(): number;
  utcOffset(b: number): Moment;
  utcOffset(b: number): Moment;
}

// ok
interface Moment {
  utcOffset(): number;
  utcOffset(b: number | string): Moment;
}

function gn(x: string): void
function gn(x: number): void
function gn(x: number | string) {
  return moment().utcOffset(x)
}
