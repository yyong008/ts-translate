# 介绍 Introduction

为了程序的可用性，我们需要一些最简单的数据单元，数字、 字符串、结构、布尔值。在 typescript 中，我们支持和 JavaScript 几乎相同的类型，提供了方便的 枚举 类型方便我们解决问题。

# 布尔类型 Boolean

在 JavaScript 和 TypeScript 中，最简单的数据类型是值 true/false，被叫做 布尔 类型。

let isDone: boolean = false

# 数字类型 Number

跟在 JavaScript 中一样的，所有数字类型在 TypeScript 中也都是浮点型的值。这些浮点型的值是 number 类型。除了十六进制 ( hexdecimal literals )和十进制(decimal literals )，TypeScript 从 ECMAScript2015 中引入也支持二进制 ( binary literals) 和十六进制 ( octal literals )。

let decimal: number = 6
let hex: number = 0xf00d
let binary: number = 0b1010
let octal: number = 0o744

# 字符类型 String

另一个基本的部分用 JavaScript 创建网页和服务是文本数据。跟其他的语言一样，我们使用 string 类型 引用文本数据。和 JavaScript 一样， TypeScript 也是使用双引号 " 或者单引号 ' 来包裹字符串类型。

let color: string = 'blue'
color = 'red'

当有多行或者有嵌入表达式， 我们也可以使用模板字符串。字符包含在反引号 (`)，嵌入表示式就是形如这样 ${ expr }。

let fullName: string = `Bob Bobbington`
let age: number = 37
let sentence: string = `Hello, my name is ${fullName}.

I'll be ${age + 1} years old next month.`

这也等价于声明像这样的语句：

let sentence: string =
  'Hello, my name is ' +
  fullName +
  '.\n\n' +
  "I'll be " +
  (age + 1) +
  ' years old next month.'

# 数组 Array

跟 JavaScript 一样， TypeScript 也允许尼使用数组。数组有两种书写的形式。第一种，我们可以在元素后面跟一个 [] 去表示这是一个数组类型。

let list: number[] = [1, 2, 3]

另外一种是用泛型数组类型，Array<elemType>

let list: Array<number> = [1, 2, 3]

# 元组类型 Tuple

元祖允许我们用已知固定的几个元素去表达一个数组，但是也不必相同。例如： 我们想用一对值去描述一个值：

// 声明一个元祖
let x: [string, number]；

// 初始化元祖 x
x = ["hello", 10]

// 错误的初始化
x = [10 ,"helle"]

当我们用下标访问一个元素的时候，元素类型已经被返回：

console.log(x[0].substring(1)) // OK
console.log(x[1].substring(1)) // Error, 'number' does not have 'substring'

访问一个已知索引范围之外的元素会失败，并提示一下的内容：

x[3] = 'world' // cuow, Property '3' does not exist on type '[string, number]'.

console.log(x[5].toString()) // Error, Property '5' does not exist on type '[string, number]'

# 枚举 Enum

枚举类型是一个对 JavaScript 标准数据类型的友好的补充。跟在 c# 中一样，枚举是一种为数字值集赋予更友好名称的方法。

enum Color {
  Red,
  Green,
  Blue,
}

let c: Color = Color.Green

默认的情况下，枚举的成员应该是从 0 开始的，我们可以手动的设置每一个成员的值。例如， 我们可以将之前的例子的 0 用 1 来替换。

enum Color {
  Red = 1,
  Green,
  Blue,
}
let c: Color = Color.Green

或者， 设置我们可以手动的设施所有的值

enum Color {
  Red = 1,
  Green = 2,
  Blue = 4,
}
let c: Color = Color.Green

一个方便的功能你可以使用一个数字值在枚举中获取一个数据值的名字。例如，我们有一个 2 但是不确定上面的 Color 在枚举中被被映射的值，我们能够正确的查看名字：

enum Color {
  Red = 1,
  Green,
  Blue,
}
let colorName: string = Color[2]

console.log(colorName) // Displays 'Green' as its value is 2 above

# Any 任意类型

当我们正在写一个应用程序我们需要描述一个我们不知道类型的变量。这个值能带来动态的内容。例如从第三方库。在这种情况下，我们想退出类型检查和能够通过编译的类型检查。这样做，我们使用 any 类型标注它们：

let notSure: any = 4
notSure = 'maybe a string instead'
notSure = false // okay, definitely a boolean

any 类型是非常有用的，在现有的 JavaScript，在编译期间允许我们，渐进的加入或者退出类型检查。但是你可能期望 Object 与其他语言一样，有类似的行为。但是，对象类型的变量仅允许您为其分配任何值。您不能在它们上调用任意方法，即使是实际存在的方法：

let notSure: any = 4
notSure.ifItExists() // okay, ifItExists might exist at runtime
notSure.toFixed() // okay, toFixed exists (but the compiler doesn't check)

let prettySure: Object = 4
prettySure.toFixed() // Error: Property 'toFixed' doesn't exist on type 'Object'.

注意：避免使用支持非原始对象类型的对象，如我们的“Do’s and Don’ts部分所述.

any 类型是方便的如果你只知道类型的一部分，并不是所有的类型。例如：我们有一个数组，但是数组呼和可不同的类型：

let list: any[] = [1, true, 'free']
list[1] = 100

# Void 空

空类型相反与 any 类型，根本没有任意类型，您可能通常将其视为无返回值的函数的返回类型：

function warnUser(): void {
  console.log('This is my warning message')
}

声明一个 void 类型的变量是没有什么用的，因为你仅仅能够分配一个 null (仅仅当没有指定 --strictNUllCheks，见下一节) 或者 undefined 给它们。

let unsable: void = undefined
usabled = null // OK if `--strictNullChecks` is not given

# Null and Undefined

在 TypeScript 中，undefined 和 null 实际上都有自己的类型，分别命名为 undefined 和 null。就像 void 一样，它们本身并不是非常有用：

// Not much else we can assign to these variables!
let u: undefined = undefined
let n: null = null

默认情况下，null 和 undefined 是所有其他类型的子类型。这意味着您可以将 null 和 undefined 分配给数字。

但是，当使用 --strictNullChecks 标志时，null 和 undefined 仅可分配给任何类型及其各自的类型（一个例外是 undefined 也可分配给 void ）。这有助于避免许多常见错误。如果要传递 string 或 null 或 undefined，则可以使用联合类型 string | void | undefined。

联合类型是一个高级主题，我们将在下一章中介绍。

注意：我们鼓励在可能的情况下使用--strictNullChecks，但是出于本手册的目的，我们将假定它已关闭。

# Never

Never 类型表示永不出现的值的类型。例如，Never 是始终抛出异常的函数表达式或箭头函数表达式的返回类型，或者是永不返回的异常；变量也永远不会被任何永远不可能为真的类型防护所缩小，从而获得类型。

never 类型是每种类型的子类型，并且可以分配给每种类型。但是，任何类型都不是（永远不会除外的）永不的子类型或可分配给它的子类型。甚至任何东西都无法分配给永不。

函数永不返回的一些示例：

// Function returning never must have unreachable end point
function error(message: string): never {
  throw new Error(message)
}

// Inferred return type is never
function fail() {
  return error('Something failed')
}

// Function returning never must have unreachable end point
function infiniteLoop(): never {
  while (true) {}
}

# 对象 Object

object 是代表非基本类型的类型，即非 number，string，loolean，bigint，symbol，null 或 undefined 的任何类型。

使用对象类型，可以更好地表示 Object.create 之类的 API。例如：

eclare function create(o: object | null): void;

create({ prop: 0 }); // OK
create(null); // OK

create(42); // Error
create("string"); // Error
create(false); // Error
create(undefined); // Error

# 类型断言

有时，您会遇到比 TypeScript 更了解值的情况。通常，当您知道某个实体的类型可能比其当前类型更具体时，就会发生这种情况。

类型断言是一种告诉编译器“相信我，我知道我在做什么的方式”。类型断言就像其他语言中的类型转换一样，但是不执行任何特殊的数据检查或重构。它对运行时间没有影响，仅由编译器使用。 TypeScript 假定您（程序员）已经执行了所需的任何特殊检查。

类型断言有两种形式。一种是“尖括号”语法：

let someValue: any = 'this is a string'

let strLength: number = (<string>someValue).length

另一种是 as 语法：

let someValue: any = 'this is a string'

let strLength: number = (someValue as string).length

这两个样本是等效的。优先使用一种是另一种选择。但是，将 TypeScript 与 JSX 一起使用时，仅允许使用 as 样式的断言。

# 关于 let 的注释

您可能已经注意到，到目前为止，我们一直在使用 let 关键字而不是您可能更熟悉的 JavaScript 的 var 关键字。 let 关键字在 ES2015 中引入 JavaScript，由于它比 var 更安全，因此现在被视为标准关键字。稍后我们将详细讨论，但是使用 let 可以缓解 JavaScript 中的许多常见问题，因此您应尽可能使用它而不是 var。
